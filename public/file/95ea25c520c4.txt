1、利用D:\MobileNet-SSD-windows-master\scripts下的build_win.cmd进行编译

   1）设置环境变量python c:\python27，对build_win.cmd中的设置项进行检查，主要是目录设置等，python的caffe接口很长时间没有编译通过，后来删除build目录下的以前编译的东西后重新进行了编译才通过。
   2) pip 安装google.protobuf pip在 c:\python27\scripts  pip install protobuf
   3) Git cmd可以执行sh脚本文件，Git 单独下载安装后使用
   4）编译caffe后，将caffe python目录下的caffe拷贝到python27\lite-packages目录下（python的caffe接口）

2、下载VOCO2007VOCO2012图像集解压到data目录下，这两个图像集合并生产VOCO0712。create_data.sh和Create_list.sh从别处下载得到的


3、执行D:\MobileNet-SSD-windows-master\data\VOC0712\create_list.sh。它产生文件列表，create_data.sh时，它会关联到create_annset.py,该文件中有两句
from google.protobuf import text_format

from caffe.proto import caffe_pb2


   这两句就是前面准备的模块。

4、在create_annset.py中有build/tools/Release/convert_annoset这一句要执行build_win.cmd编译后的build目录下的convert_annoset.exe,将编译后的build拷贝到
    
   D:\MobileNet-SSD-windows-master目录下。原来是在D:\MobileNet-SSD-windows-master\scripts目录下，这个问题导致很长时间create_annset.py没有执行成功。同时原来的目录build/tools/Release/convert_annoset中没有Release这个，
   
   在执行生成create_data.sh脚本时，总报windowserror[error2],意思是没找到相关文件。注意360杀毒将exe隔离的问题。



5、在执行create_annset.py过程中，出现缺少python下的库问题，用pip进行安装即可。

6、在create_annset.py执行后，出现symlink的报错，这个在windows下没有，先别管他。




自己数据训练步骤：

1、利用下载的图片加工工具后生产图像标注的lable的xml文件，工具在C:\Users\LIUZH\Downloads\windows_v1.7.0\windows_v1.7.0下面，快捷键w，d和空格，w出现十字架，d切换下一张，空格保存加工的图片
2、修改标签的种类名称，需要利用F：盘的车牌样本数据加工下面的changelable.py工具进行修改。
3、将数据拷贝到D:\MobileNet-SSD-windows-master\data\VOC2012下面，按照目录分别放好。
4、利用D:\MobileNet-SSD-windows-master\scripts目录下面的txt2.py产生VOC2012下面main目录下的四个文件。
5、修改D:\MobileNet-SSD-windows-master\data\VOC0712下面的labelmap_voc.prototxt中的类别标签
6、利用D:\MobileNet-SSD-windows-master\data\VOC0712下的create_list.sh在Git Cmd窗口中执行，如果没有生产test和trainval文件，查看shell文件中dataset in test trainval,可以修改生成两个文件。
7、利用D:\MobileNet-SSD-windows-master\data\VOC0712下的create_data.sh在Git Cmd窗口中执行,产生lmdb文件。并将D:\MobileNet-SSD-windows-master\data\VOCdevkit\VOC0712\lmdb\VOC0712_trainval_lmdb
   下面的数据拷贝到exmple目录下一份。D:\MobileNet-SSD-windows-master\examples\VOC0712\VOC0712_trainval_lmdb
8、查看D:\MobileNet-SSD-windows-master\train_mobilenet_v2.cmd中的相关文件需要修改prototxt中的类别数量
9、对比D:\MobileNet-SSD-windows-master\models\MobileNetV2下的train_voc和train_voc-1用文本比较工具，修改相应的数。按照类别种类进行修改。
10、执行D:\MobileNet-SSD-windows-master\train_mobilenet_v2.cmd进行训练，loss降低到1.5-2.5之间即可。
11、训练完测试需要修改deploy.prototxt文件中的数，仔细核对。
